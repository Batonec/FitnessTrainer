# Задача для Claude Design: карточка «Совет тренера» (iOS)

## Что нужно
Спроектировать SwiftUI-карточку с LLM-рекомендацией следующей тренировки. Карточка
ставится **вверху экрана «Тренировка» (TodayScreen), над секцией «План тренировки»**.
Нужны макеты **всех состояний** (см. ниже).

## Контекст приложения
- iOS 17, **SwiftUI**, без сторонних библиотек. Язык интерфейса — **русский**. Веса — в килограммах.
- Эстетика — «Liquid Glass»: уже есть модификатор `.glassCard(radius:)` (полупрозрачные
  стеклянные карточки со скруглениями) — **карточка должна совпадать по стилю** с
  существующими (`TodayExerciseCard`, `HistoryCard`).
- Шрифт — **JetBrains Mono** (Regular/Medium/SemiBold/Bold). Поддержать тёмную/светлую
  тему и Dynamic Type.

## Данные, которые приходят с бэкенда (`GET /api/recommendations/next`)
```json
{
  "status": "none | pending | ready | failed",
  "stale": false,                         // true = есть тренировка новее, чем та, по которой считали
  "based_on_workout_count": 56,           // по скольки тренировкам построено
  "model": "claude-opus-4-8",
  "error": null,                          // текст ошибки при status=failed
  "recommendation": {
    "focus": "короткий заголовок тренировки",
    "load_type": "heavy | medium | light",
    "rationale": "развёрнутый текст: почему такой план, а не иной (может быть длинным, 3–6 предложений)",
    "exercises": [
      { "name": "Жим ногами",
        "note": "короткое пояснение по весу/повторам относительно прошлого раза",
        "sets": [ { "reps": 12, "weight": 90 } ] }
    ]
  }
}
```

## Реальный пример (status=ready), на нём и рисуй
- **focus:** «Сбалансированная тренировка верх+низ, средняя нагрузка после перерыва»
- **load_type:** `medium` · построено по 56 тренировкам
- **rationale:** «С последней тренировки прошло 13 дней — это большой перерыв, поэтому не
  стоит сразу прыгать на максимальные веса из мая (120кг жим ногами, 80кг тяга). Последняя
  тренировка 29 мая была явно разгрузочной и прошла легко. Сейчас разумно зайти на средней
  нагрузке… Не выбрал heavy, потому что после 13 дней простоя резкий скачок повышает риск
  травмы — лучше качественная средняя сессия, затем вернуться к прогрессии.»
- **exercises** (название · подходы · note):
  1. Жим ногами — `90 кг × 12 × 3` — «между разгрузочными 60 и пиковыми 120 после перерыва»
  2. Жим в тренажере — `60 кг × 12, 12, 10` — «ниже пиковых 67.5, мягкий вход»
  3. Тяга верт. — `70 кг × 12 × 3` — «между разгрузочными 60 и рабочими 75–80»
  4. Дельты — `20 кг × 12, 12, 10` — «комфортный рабочий вес»
  5. Бицепс — `15 кг × 13, 12, 12` — «между лёгкими 10 и рабочими 20»
  6. Трицепс — `15 кг × 14 × 2` — «привычный рабочий вес»

> Формат подходов: одинаковые группировать (`90 кг × 12 × 3`), разные — перечислять
> (`60 × 12, 12, 10`).

## Состояния для макетов (5)
1. **ready** — основной вид: шапка «Совет тренера» + иконка-искра; `focus` крупно; **чип
   нагрузки** с цветовым кодом (heavy — тёплый/красный, medium — янтарный, light — спокойный
   зелёный); список упражнений (название + компактные подходы + `note` мелким приглушённым);
   блок `rationale` **сворачиваемый** («Почему так» → раскрыть); внизу две кнопки —
   **«Применить в план»** (primary) и **«Обновить»** (secondary).
2. **pending / loading** — «ИИ обновляет план…» со спиннером; допускается показывать поверх
   предыдущей рекомендации (затемнение).
3. **failed** — мягкая ошибка + кнопка **«Повторить»** (текст из `error`).
4. **none / empty** — пусто: приглашение «Сгенерировать совет» (кнопка).
5. **stale = true** — поверх ready ненавязчивый бейдж «Обновить — есть новая тренировка».

## Взаимодействия
- **«Обновить»** → запрос генерации, ~15–20 с (спиннер на кнопке/карточке).
- **«Применить в план»** → переносит упражнения в черновик тренировки.
- Тап по «Почему так» → разворачивает/сворачивает `rationale`.
- (Опц.) тап по упражнению → детали.

## Ограничения и что отдать
- Соответствие существующим карточкам (`.glassCard`, скругления, JetBrains Mono, палитра приложения).
- Компактность: подходы в одну строку, `note` — вторичный текст.
- **Отдать:** визуальный макет карточки во всех 5 состояниях (артефакт/мокап), и/или готовый
  SwiftUI-`View` под существующий дизайн.
